## Описание работы
- Установка и настройка .NET SDK
- Создание минимального веб-приложения
- Принципы маршрутизации HTTP-запросов
- Работа с middleware
- Возврат JSON-данных из API
- Защита API с помощью ключа доступа
## Главные выводы
1. ASP.NET Core — мощный и производительный фреймворк для создания веб-приложений и API
2. Middleware — ключевая концепция обработки запросов
3. Minimal API предоставляет лаконичный способ создания веб-серверов
## Итоговая таблица
| Характеристика | ASP.NET Core |
|----------------|--------------|
| Создание сервера | `WebApplication.CreateBuilder()` |
| Запуск | `app.Run()` |
| Маршрут GET | `app.MapGet("/", fn)` |
| Параметр маршрута | `{name}` → `(string name)` |
| Возврат JSON | `return new {...}` / `Results.Ok(...)` |
| Middleware | `app.Use(async (ctx, next) => ...)` |
| Логирование | Встроенное + `Console.WriteLine` |
| Статус ответа | `Results.NotFound(...)` |
| Тип данных | Строгие (C# record, class) |
## Интересные факты
ASP.NET Core используется на крупных проектах:
- **Stack Overflow** — один из крупнейших сайтов вопросов и ответов для программистов, полностью перешёл на ASP.NET Core
- **Microsoft Teams** — бэкенд для некоторых сервисов использует ASP.NET Core
- **ВКонтакте** — часть микросервисов работает на ASP.NET Core
## Ответы на вопросы
1. Minimal API это более простой и легковесный подход. MVC это полноценный фреймворк, требующий разделения на модели.
2. Middleware не будет применён к этому маршруту
3. По умолчанию используется политика именования `camelCase`, что соответствует общепринятым стандартам JavaScript/JSON
4. 
- 200 — это код успешного выполнения запроса. 
- 401 — означает, что запрос требует аутентификации.
- 404 — сервер не может найти запрошенный ресурс по указанному URL. 
5. 
- dotnet run — однократно запускает приложение. 
- dotnet watch run — запускает приложение в режиме наблюдения за файлами. 
## Таблица
| Команда / метод | Что делает |
|-----------------|------------|
| `dotnet --version` | Проверить версию .NET SDK |
| `dotnet new web` | Создать минимальное веб-приложение |
| `dotnet run` | Запустить сервер |
| `dotnet watch run` | Запустить с автоперезагрузкой при изменениях |
| `dotnet restore` | Восстановить пакеты/зависимости |
| `dotnet new gitignore` | Создать .gitignore для .NET-проекта |
| `app.MapGet(path, fn)` | Зарегистрировать GET-маршрут |
| `app.Use(middleware)` | Добавить middleware в цепочку |
| `Results.Ok(data)` | Вернуть ответ 200 с данными |
| `Results.NotFound(data)` | Вернуть ответ 404 с данными |
